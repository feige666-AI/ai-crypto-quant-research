# 作品集讲解

## 项目解决什么问题

很多量化学习项目只有一段策略代码，别人很难确认数据是否可靠、结果如何产生、命令能否复现。这个项目把基础研究流程拆成可运行的小模块，让招聘人员可以从输入 CSV 一直追到交易明细、指标和图表。

## 运行后会得到什么

`python -m crypto_quant_research demo` 会校验 300 行合成 OHLCV，生成均线信号，运行含手续费和滑点的只做多回测，并输出信号、资金曲线、交易明细、JSON 摘要和三张图表。

## 项目如何拆分

- 数据模块只负责读取和校验。
- 策略模块只负责均线和信号。
- 回测模块只负责资金、持仓、费用和指标。
- 报告和图表模块只负责表达结果。
- CLI 负责把这些步骤串起来。

这种拆分让每个模块都容易测试，也避免把数据问题、策略观点和资金计算混在一起。

## 我完成了什么

- 确定项目定位、范围和“不连接交易所”的安全边界。
- 检查原始仓库与输出，指出安装、校验、回测、文档和隐私问题。
- 提出清晰验收条件，运行测试并检查最终图表和报告。
- 整理实际完成内容，避免把计划写成成果。

## Codex 如何参与

Codex 辅助审计仓库、拆解任务、生成和修改代码、编写测试、排查错误、运行命令和整理文档。人工负责目标、范围、风险判断、输出检查和是否接受结果。

这不是“全部独立手写”，也不是简单复制粘贴。关键能力在于把需求变成可验证标准，并用测试、覆盖率、代码质量工具和人工检查判断生成结果是否可信。

## 项目体现的能力

- Python 标准库和模块化设计。
- CSV 数据质量检查和错误信息设计。
- 简化交易规则、PnL、回撤和统计指标计算。
- pytest 边界测试、覆盖率、Ruff 和 CI。
- 面向技术人员与非技术招聘人员的文档表达。

## 限制

项目只使用合成数据、单资产和只做多模型。当前收盘价产生信号并成交是教学简化，不能证明真实策略有效，也不能用于实盘。
