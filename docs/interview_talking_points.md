# 面试讲解要点

## 30 秒介绍

这是一个本地 Python 量化研究学习 Demo。我把合成 OHLCV 从严格 CSV 校验、均线信号、含手续费和滑点的只做多回测，一直做到交易明细、指标、图表、77 个测试和 GitHub Actions。项目不接交易所，重点是展示可复现、可解释和可验证的工程流程。

## 2 分钟介绍

原始版本只能手动设置 `PYTHONPATH` 运行一个简化回测，数据校验和指标都很有限。我先记录基线，再把它整理成标准 `src` 包，增加 `validate`、`signals`、`backtest`、`demo` 四个命令。数据层会拒绝缺字段、NaN、Infinity、负价格、错误 OHLC、重复或乱序时间；策略层只保留简单均线；回测层明确成交价、手续费、滑点、期末仓位、基准、回撤、暴露率、胜率和 Profit Factor。最后用固定种子生成 300 行合成数据，通过 77 个测试、96% 覆盖率、Ruff、编译检查和三版本 CI 验证。

## 常见技术问题

### 为什么使用均线策略？

它容易解释，能产生明确交叉事件，适合把重点放在数据、执行、指标和测试，而不是复杂指标堆砌。

### 为什么只做多？

作品集目标是基础流程。做空会引入借贷、保证金、强平和不同收益结构，增加复杂度但不帮助说明核心能力。

### 为什么使用合成数据？

固定随机种子让结果可重复，不涉及付费数据、授权、账户或真实交易隐私。代价是不能据此判断真实市场表现。

### CSV 如何校验？

先检查路径、普通文件、非空和表头，再逐行检查必需字段、有限浮点数、正价格、非负成交量、OHLC 关系、时间非空、重复和升序；错误会带文件、行号、字段和值。

### 最大回撤如何计算？

每一行维护历史最高权益，当前回撤是 `(峰值权益 - 当前权益) / 峰值权益`，所有行中的最大值就是最大回撤。

### 手续费和滑点如何处理？

买入成交价高于收盘价，卖出成交价低于收盘价；两端手续费都按成交金额计算。全仓买入的数量会预留入场手续费。

### 胜率如何计算？

盈利完成交易数除以全部完成交易数。未平仓不算完成交易；没有完成交易时输出 `null`。

### Profit Factor 如何计算？

盈利交易净利润总和除以亏损交易净亏损绝对值。没有亏损交易时输出 `null`，因为用 0 或一个巨大数字会误导。

### 未平仓如何处理？

默认末尾强制平仓并记录 `end_of_data`。关闭后按末价计权益，标记 `open_position=true`，不计入完成交易。

## 遇到的问题

- 原始 README 命令围栏损坏，并依赖手动 `PYTHONPATH`。
- 原始回测把动作数当交易数，没有滑点和完整交易统计。
- Git 克隆通道在当前环境被网络代理拒绝，因此使用 GitHub 官方 API 同步基线并发布真实分支提交。
- matplotlib 默认缓存目录不可写，代码改用进程临时目录，避免用户环境警告。

## 面试官可能追问

- 为什么当前收盘成交可能有偏差？
- 如何证明没有未来函数？
- 费用为何影响仓位数量而不是让现金为负？
- 为什么没有亏损时 Profit Factor 是 `null`？
- 如果换成真实历史数据，如何避免过拟合？

回答时应客观说明简化模型、数据边界和验证方法，不把合成结果包装成策略优势。
