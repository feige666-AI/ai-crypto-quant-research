# DECISIONS

## 2026-05-13：使用本地 CSV 作为第一阶段数据来源

决策：当前版本使用 `data/sample_data.csv` 作为演示输入。

原因：

- 本地 CSV 更容易解释，适合 HR 和非技术读者理解。
- 不需要 API Key，避免泄露交易所账号、Token 或真实账户信息。
- 对学习量化流程来说，先跑通“读取数据 → 生成信号 → 输出结果”比直接接 API 更稳妥。

取舍：

- 优点是简单、安全、可复现。
- 缺点是数据量小，不能代表真实市场研究结论。

## 2026-05-13：先做本地回测，不做实盘交易

决策：当前项目只做本地回测 Demo，不接入实盘交易。

原因：

- 实盘交易涉及资金风险、账户安全和交易所规则，不适合作为第一阶段作品集功能。
- 本项目定位是学习和验证量化研究流程，不是投资工具。
- 对求职展示来说，清晰的项目流程和文档比“看起来很复杂的实盘功能”更重要。

## 2026-05-13：当前先使用 argparse，暂不引入 Typer

决策：当前版本使用 Python 标准库 `argparse` 做命令行入口。

原因：

- 当前命令只有一个 `backtest`，标准库已经足够。
- 不引入第三方依赖，能降低安装和运行门槛。
- 后续如果命令变多，再迁移到 Typer 会更合理。

后续计划：

- 当项目增加 `fetch-data`、`analyze-result`、`plot-equity` 等多个命令时，可以引入 Typer，让 CLI 更清晰。

## 2026-05-13：示例数据和结果只用于展示

决策：仓库只保留小型样例数据和样例结果。

原因：

- 避免上传真实交易记录、真实账户数据和大体积原始数据。
- 让 HR 或面试官打开仓库后能快速看到流程。
- 大文件和私有数据应放在 `data/raw/`、`data/private/` 或 `results/private/`，并由 `.gitignore` 忽略。
